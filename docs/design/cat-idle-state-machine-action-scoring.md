# 猫咪 idle 状态机：实际计分规则

本文是当前网页端 Cat Mind 的实际数值说明，不记录历史方案。用于看调试面板、调整体验和核对代码。

## 1. 怎么读分数

- 五维在代码内部保存为 `0–1`，本文和调试面板统一显示为 `0–100`。
- 每轮先检查硬锁、tier 和 provider；不满足时不能执行。
- 可执行动作的最终分：`基础分 - 自己的冷却扣分`。
- 分数达到动作阈值才是候选；候选中最高分获胜。同分使用固定顺序，不掷随机数。
- `quiet` 和 `stay_idle` 没有自己的分数：硬锁时是 `quiet`；没有动作达到阈值时是 `stay_idle`。

调试面板会把“理论分（基础分－冷却）”与“本轮可用分（已通过动作条件）”分开显示；理论分不代表当前能执行。

## 2. 五维

| 字段 | 含义 | 主要影响 |
|---|---|---|
| `appetite` | 食欲 | 吃零食；高时轻微压低移动、玩球 |
| `sleepiness` | 困意 | CAT2 / CAT3 睡眠反馈；压低活跃动作 |
| `energy` | 精力 | 小移动、玩球；低时压低轻声回应 |
| `social_need` | 社交需求 | 轻声回应，也轻微推动移动、玩球、吃 |
| `stimulation_need` | 刺激需求 / 无聊 | 小移动、玩球；轻微推动轻声回应 |

### 进入猫形态的初始值

| 进入方式 | 食欲 | 困意 | 精力 | 社交 | 刺激 |
|---|---:|---:|---:|---:|---:|
| 手动 | 22 | 12 | 75 | 22 | 28 |
| 自动 | 32 | 22 | 66 | 32 | 42 |

### 每分钟自然变化

| tier | 食欲 | 困意 | 精力 | 社交 | 刺激 |
|---|---:|---:|---:|---:|---:|
| CAT1 | +0.55 | +0.45 | -0.45 | +2.4 | +3.2 |
| CAT2 | +0.45 | +1.35 | -0.8 | +1.0 | +0.65 |
| CAT3 | +0.3 | +0.85 | -0.35 | +0.55 | +0.3 |

时钟每 30 秒检查一次；实际按经过时间结算，因此后台暂停后恢复时不会少算。

## 3. 哪些动作能被选中

| 动作 ID | 用户看到的行为 | tier | 阈值 | 冷却 | provider 必须确认 |
|---|---|---|---:|---|---|
| `cat1_social_ping` | 轻声回应 / 气泡 | CAT1 | 45 | 180 秒 | 当前可显示和播放，且不在既有表现锁中 |
| `cat1_small_move` | 猫和聊天球的小幅移动 | CAT1 | 54 | 70 秒 | 已在聊天球附近、有移动空间、当前 settled |
| `cat1_play_yarn` | 玩毛线球 | CAT1 | 54 | 240 秒 | 已在聊天球附近，毛线球可隐藏和恢复 |
| `cat1_eat_snack` | 吃零食 | CAT1 | 52 | 180 秒 | 既有吃东西 runner 可用 |
| `cat2_nap_feedback` | 小憩反馈 | CAT2 | 50 | 240 秒 | 睡眠声音 / 气泡能力可用 |
| `cat3_sleep_feedback` | 熟睡反馈 | CAT3 | 45 | 300 秒 | 睡眠声音 / 气泡能力可用 |

以下情况一律输出 `quiet`：return、拖拽准备或拖拽中、CAT1 贴边半隐藏、切换中、已有独立动作执行中、return ball 不可见、猫运行态无效、聊天面板正在拖拽。

“靠近聊天球”是小移动和玩球的硬条件。猫远离聊天球时，只能等待原有表现把它带近；状态机绝不主动发起 walk-to-chat。

冷却只在 runner 已真正 `started` 后写入。provider 拒绝、启动前失败、取消或打断都不会制造冷却。

## 4. 基础分公式

下列变量均使用 `0–100` 的显示值。每一项只按基础分与自己的冷却扣分计算；没有额外的 recent、拖拽或气泡加减分层。

| 动作 | 基础分 |
|---|---|
| 轻声回应 | `12 + 社交×0.55 + 刺激×0.08 + (100-困意)×0.04` |
| 小移动 | `18 + 刺激×0.40 + 精力×0.24 + 社交×0.06 - 困意×0.10 - 食欲×0.05` |
| 玩毛线 | `12 + 刺激×0.48 + 精力×0.24 + 社交×0.12 - 困意×0.12 - 食欲×0.08` |
| 吃零食 | `18 + 食欲×0.55 + (100-精力)×0.06 + 困意×0.04 + 社交×0.06 - 刺激×0.05` |
| CAT2 小憩 | `12 + 困意×0.55 + (100-精力)×0.22 - 刺激×0.08 + 食欲×0.03 - 社交×0.04 + 8` |
| CAT3 熟睡 | `12 + 困意×0.55 + (100-精力)×0.22 - 刺激×0.08 + 食欲×0.03 - 社交×0.04 + 20` |

### 冷却扣分

冷却不是硬禁止。动作一开始会承受最大扣分，随剩余冷却时间线性减弱；到期为 0。每个动作只扣自己，不影响其他动作。

| 动作 | 刚开始时扣分 | 到期前最低扣分 |
|---|---:|---:|
| 轻声回应 | 16 | 2 |
| 小移动 | 20 | 6 |
| 玩毛线 | 28 | 12 |
| 吃零食 | 20 | 8 |
| CAT2 小憩 | 24 | 10 |
| CAT3 熟睡 | 26 | 12 |

## 5. 事件如何改变五维

### 用户互动

| 事件 | 实际变化 |
|---|---|
| `drag_start` | 社交 +4，刺激 +5，精力 -2 |
| `drag_end` | 社交 +8，刺激 +8，精力 -4 |
| `drag_cancelled` | 社交 +3，刺激 +2 |
| `rapid_drag` | 社交 +12，刺激 +12，精力 -10，困意 +5 |
| `thought_bubble_pop` | 社交 +6，刺激 +4，食欲 +4 |
| `cat_hover_reaction` | 社交 +5，刺激 +3，精力 -1 |

所有用户主动互动都会提高既有的社交或刺激需求：用户越持续地摸、点、拖，合法动作的基础分越高。它们仍不会直接启动吃东西或轻声回应，且 hard gate、provider 与每个动作的 cooldown 继续生效。

### 现有表现和桌面 observation

桌面聊天窗口的 `reason: "poll"` 是存活心跳，不是新的桌面经历。Cat Mind normalizer 必须以当前最小化状态和窗口位置/尺寸为准去重：首次观察到、最小化状态改变或位置/尺寸确实改变时，才生成下表对应 observation、更新五维并排入下一轮判断；原生 IPC、BroadcastChannel 或本地 UI 重复送达的同状态/同 rect 一律忽略，不能累加五维、recent events 或判断机会。`idle-dock-enter` 独立记录一次，直到窗口离开该落位。

|---|---|
| `chat_minimized_visible` | 社交 +3，刺激 +7 |
| `chat_minimized_moved_far` | 社交 +4，刺激 +10 |
| `chat_compact_surface_visible` | 社交 +3，刺激 +8 |

## 6. 排序与正常等待

1. 先移除不符合 hard gate、tier 或 provider 条件的动作。
2. 只对 provider 已允许的动作计算本轮可用分。
3. 移除可用分低于自身阈值的动作。
4. 同分固定顺序：轻声回应 → CAT2 小憩 → CAT3 熟睡 → 小移动 → 吃零食 → 玩毛线。
5. 没有候选时保持 `stay_idle`，这不是失败，也不会为了填空而播放气泡。
