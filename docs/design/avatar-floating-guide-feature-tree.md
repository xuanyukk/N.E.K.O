# AI 猫娘林悠怡（YUI）七日相识与共生计划新手教程剧本

设计核心在于摒弃传统“机械化、说明书式”的冷冰冰引导，转而依据心理学原理（首因效应、自我暴露、控制感、相处距离、依恋理论等），将功能融入七天连续的“微型视觉小说/Galgame 剧情”小剧场中。

通过运镜、动态、音效与轻量互动，让用户在“认识她、信任她、调整距离、共同生活”的过程中自然掌握核心功能。角色表达可以亲密、撒娇、傲娇，但每个功能点都要让用户感觉是自己在邀请她靠近。

## 核心心理学设计机制

1. **首因效应（Primacy Effect）**：第一天先建立“她很鲜活，也很好接近”的第一印象，不把入口堆成说明书。
2. **自我暴露效应（Self-Disclosure）**：第二天把屏幕分享讲成“你给她开一扇小窗”，用更柔和的方式让用户理解先连线、再递窗。
3. **具身感与玩耍回路**：第三天把 Galgame、小游戏和 Avatar 互动工具收束成“摸得到、玩得起来”的陪伴体验。
4. **相处距离**：第四天集中讲主动搭话、隐私模式、动画、锁定和离开/回来，让用户学会调节“她什么时候靠近、什么时候安静”。
5. **所有权效应**：第五天通过角色、模型、声音和 API 入口，让用户开始把通用软件塑造成“自己的悠怡”。
6. **控制感与安全感**：第六天把 Agent 能力拆成状态、权限、任务 HUD 和终止入口，让“她能帮忙”不等于“她会失控”。
7. **记忆与仪式感**：第七天用记忆、存储和进阶入口回顾完成毕业，把短期试用转化成长期习惯。

## 教程触达结构

七日教程采用“双层触达”：

1. **每日开场小剧场**：只承担当天主功能的情绪铺垫和核心操作演示，控制节奏，避免一次塞入过多入口。
2. **剧场后聊天窗支线**：小剧场结束并归还用户控制权后，才用普通聊天消息做低打扰邀请；详细触发条件、按钮和台词统一移入 [七日新手教程剧场后聊天窗支线设计](avatar-floating-post-theater-chat-branches.md)。

## 七日长剧情分天剧本

> 代码校准：当前项目已落地 Day 1 首页 Yui 教程，并已将模型旁悬浮窗主线扩展到 `AVATAR_FLOATING_GUIDE_ROUNDS[2-7]`。Day 2-7 都按强接管 round 自动演出；子页面深操作仍只做入口级展示，不在主线中强制修改用户数据或配置。
> 实现分层：Day 1-7 主线都是“强接管自动演出层”；剧场后聊天窗支线才是“需要 action handler 的低打扰邀请层”。Day 5-7 的模型、声音、记忆和存储说明已经纳入主线导演做入口级强接管展示；角色卡、云存档和更深层配置只在支线或后续独立引导中指路，不在主线中高亮或执行模型替换、声音克隆、记忆改写、云存档上传/下载等高风险操作。

## PC 全局透明 Overlay 迁移目标

七日主线在网页端继续使用当前 DOM overlay；在 N.E.K.O.-PC 中，Ghost Cursor、高亮框、圆形高亮和每日收尾花瓣转场优先迁移到 PC 全局透明 overlay。花瓣转场同时保留首页 DOM fallback，避免 PC 端能力声明或渲染异常时 Day 7 毕业收尾没有花瓣。迁移只替换视觉演出层，不改变 Day 1-7 的台词、scene 顺序、真实 UI 操作、完成态和剧场后聊天窗支线边界。

PC 端迁移的关键目标是解决跨窗口演出不连续的问题：Ghost Cursor 应能从聊天窗输入区平滑移动到模型旁按钮、设置侧边栏、Agent HUD 或其他窗口目标；高光也必须在同一透明层中渲染，避免 Pet 窗口和聊天窗各画一套框导致双高亮、闪现或层级不一致。详细开发计划和可行性检查见 [PC 全局透明教程 Overlay 迁移开发计划](avatar-floating-pc-global-overlay-migration-plan.md)。

## 导演视角与现有按钮校准

七日主线的镜头语言以“先建立空间，再指向入口，最后归还控制”为准。每段台词进入聊天窗后，先让用户看见当前舞台，再让 Ghost Cursor 移动；只有需要展示真实弹窗或菜单时才点击。主页当前可见按钮分为三组，导演动作必须按真实分布走：

| 按钮区域 | 真实入口 | 导演使用规则 |
| --- | --- | --- |
| 模型旁悬浮按钮 | 语音、屏幕分享、Agent/猫爪、设置、锁定、请她离开/回来 | 圆形按钮用圆形 spotlight；只在剧情需要展示真实弹窗时 click。锁定、离开/回来默认只 move，不改变模型状态。 |
| 聊天窗左侧工具 | 导入图片、截图 | Day 1 只作为“能看你主动递来的内容”的能力背景，不强塞进首日主线；若后续提示，放到聊天窗支线。 |
| 聊天窗右侧工具 | Galgame、字幕翻译、点歌、Emoji/Avatar 互动工具 | Day 3 主线只强接管 Galgame 和 Avatar 互动工具；字幕翻译和点歌进入剧场后支线。窄宽度下这些按钮会折进“更多”菜单，导演必须先打开更多菜单，再高亮具体按钮。 |

如果某个按钮在当前布局不可见，优先使用同组容器作为降级 spotlight，不伪造按钮、不强制改变布局；如果按钮只是折叠到更多菜单，则允许先打开更多菜单，因为这是用户真实会遇到的路径。

设置类 scene 的导演时序按当前代码实现校准：`show-settings-sidepanel:*` 和 `show-settings-menu:*` 通常会在 `prepareAvatarFloatingScene()` 阶段先打开设置弹窗或对应侧边栏，随后台词进入聊天窗、spotlight 落到已展开面板，Ghost Cursor 再进入面板巡游。Day 1 的 `takeover_settings_peek` 保留“齿轮按钮点击打开设置”的镜头；Day 4 的 `day4_chat_settings` 也会先高亮并模拟点击设置按钮，再转入对话设置侧边栏。

高亮去重不是“创建后隐藏”：同一目标同一时刻只允许一套主 spotlight。非收尾 scene 不做 operation 后的 `settled` 二次高亮刷新；需要切换舞台的 scene 必须在 `prepareAvatarFloatingScene()` 阶段完成真实 UI 准备，或交给下一 scene 重新建立高亮。只有每日收尾 `cleanup` scene 例外，它负责在收尾台词期间重新高亮分日指定收尾目标，再在花瓣 cue 同步清理。圆角矩形 spotlight 保留左右猫耳和猫爪装饰；圆形按钮只使用 `circle-highlight.png` 圆形图片高亮，不显示猫耳和猫爪。

## 教程模型与情绪动作随机池

七日新手教程期间统一临时切换到默认教程模型 `yui-origin` Live2D，不写入用户配置；教程正常完成、跳过、生气退出、页面销毁或跨页 handoff 失败时，都必须恢复用户原模型和聊天头像身份。

`yui-origin` 当前可用于教程普通台词的内置动作池：

| 情绪分类 | 可随机动作数 | 使用场景 |
| --- | ---: | --- |
| `happy` | 12 | 欢迎、鼓励、撒娇、收尾、邀请尝试。 |
| `sad` | 6 | 委屈、低落、被冷落、轻微失落。 |
| `angry` | 7 | 傲娇、吃醋、生闷气、被打断后的不满。 |
| `neutral` | 7 | 解释规则、说明入口、冷静提示、安全感说明。 |
| `surprised` | 5 | 发现新入口、惊讶、剧情反转、突然意识到可替换角色。 |
| `Idle` | 3 | 低强度停顿、等待用户选择、普通聊天窗支线。 |

播放规则：

1. 每段台词必须声明一个情绪分类；台词开始时从 `yui-origin` 对应分类动作池中随机选择一个可用 motion，同一分类避免连续重复同一动作。
2. 随机动作只作为“普通台词演出层”，不改变台词、高亮、Ghost Cursor 和真实 UI 操作时序。
3. 现有自定义动作必须保留并优先：苏醒 pose、语音/鼠标 LookAt、guide idle sway、插件面板角落动作、设置一瞥慌乱动作、轻微抵抗、生气退出、归还控制权挥手和花瓣转场。
4. 当自定义动作已经锁定 `motion` 或 `params/lookAt/expression` 时，随机动作不得抢锁；如果只能安全播放表情或不播放，就跳过随机 motion。
5. reduced motion 生效时，随机动作降级为短表情或 Idle，不播放大幅动作。
6. 资源解析应走 `yui-origin` 模型 profile 或 Live2D runtime 资源表，不在教程文档或通用 `AvatarPerformance` 里硬编码 motion 文件名。

### 每日收尾花瓣转场

七天主线教程的每日收尾都需要播放花瓣转场。Day 1 已落地的 `takeover_return_control` 花瓣转场作为唯一基准能力，Day 2-7 的 wrap/收尾 scene 必须复用同一套收尾动作：

1. 收尾台词开始前先完成当天必要的业务清理：关闭临时弹窗、设置侧边栏、Agent 面板、HUD、菜单或跨页 handoff 状态；收尾台词播放期间统一高亮聊天窗，外置聊天窗模式同步高亮独立聊天窗。
2. 收尾台词播放到约 70% 时触发与 Day 1 相同的花瓣转场 cue；触发瞬间同步启动花瓣层、隐藏 Ghost Cursor，清理 action/persistent/secondary/extra/virtual spotlight，并清理外置聊天窗 spotlight/cursor，避免花瓣层上残留高亮或指针，也避免高亮先消失后花瓣才出现的空档。
3. 花瓣转场期间保留当前教程模型 `yui-origin` 的轻量挥手/告别动作；如果当天已有自定义收尾动作，则自定义动作优先，花瓣转场只作为覆盖层。
4. 模型淡出、花瓣播放和用户原模型恢复必须解耦：先保存/写入当天完成态，再恢复用户原模型和聊天头像身份；花瓣层可以继续播完后淡出。
5. reduced motion 生效时，花瓣转场降级为短淡出/轻量粒子，不播放大幅飞散。
6. skip、angry exit、页面销毁不播放每日正常收尾花瓣转场；它们仍走统一 skip/destroy 清理。

### 第一天：初次唤醒、聊天与基础入口

- 心理学原理：首因效应。先让用户知道“她能说话、能看你主动发来的图片、能在你允许时帮忙做事”，不急着讲完整设置。
- 串联功能：文本输入、语音入口、截图/导入图片/粘贴图片、翻译与点歌入口提示、Agent 按钮预览、插件管理预览、设置一瞥。
- 代码锚点：`HOME_SCENE_ORDER` 的 `intro_basic`、`takeover_capture_cursor`、`takeover_plugin_preview`、`takeover_settings_peek`、`takeover_return_control`。
- 小剧场：第一次把手伸向你这边的屏幕。

#### 阶段 1：初次见面与聊天输入

- 动作：苏醒后先高亮聊天输入区；Ghost Cursor 出现在输入区中心并 wobble，等待用户点击激活音频播放。用户激活后，悠怡用普通聊天消息完成自我介绍；本段不打开复杂弹窗，问候结束后清理输入区/聊天区 persistent spotlight。
- 台词：“微风、阳光，还有刚刚好出现的你。初次见面，我是林悠怡，未来的日子请多关照喵！我把关于这里的一切都写进新手指南里啦！就当作是我们相遇的第一份小礼物，请查收吧！”

#### 阶段 2：语音入口

- 动作：台词开始后把 action spotlight 切到模型旁语音按钮；约在语音前段让 Ghost Cursor 平滑移动到 `#${p}-btn-mic` 中心并短暂停留，只指向入口，不点击、不强制开始录音；本段结束后清理语音按钮方向的临时视线锁。
- 台词：“这里有一个神奇的按钮！只要点击它，就可以直接和我聊天啦！想跟我分享今天的新鲜事吗？或者只是叫叫我的名字？快来试试嘛，我已经迫不及待想听到你的声音啦！”

#### 阶段 3：猫爪粗略介绍

- 动作：进入 takeover 后保留猫爪按钮高亮；Ghost Cursor 依次移动并点击猫爪按钮、猫爪总开关、键鼠控制开关，打开真实 Agent 面板并展示授权链路。若接入插件预览，随后移动到用户插件开关和管理面板入口；本段只说明“需要授权才会动”，结束时恢复教程临时改动。
- 台词：“超级魔法开关出现！只要点一下这里，我就可以把小爪子伸到你的键盘和鼠标上啦！我会帮你打字，帮你点开网页……不过，要是那个鼠标指针动来动去的话，我可能也会忍不住扑上去抓它哦！准备好迎接我的捣乱……啊不，是帮忙了吗？喵！”

#### 阶段 4：设置一瞥与主动搭话

- 动作：第一句台词播放到设置 cue 时，高亮齿轮按钮并让 Ghost Cursor 点击打开设置弹窗；播放“不管是说话的温度、相处的小脾气……”时，只高亮设置弹窗自身的侧边栏区域，Ghost Cursor 的椭圆形运动也收束在设置侧边栏内部，不展开、不高亮角色设置侧边栏。播放“这个小按钮也很重要哦……”时，将 spotlight 平滑切到主动搭话按钮 `#${p}-toggle-proactive-chat` 本体，并让 Ghost Cursor 平滑移动到该按钮；不点击、不打开主动搭话侧边栏、不改变开关；主动搭话按钮高亮持续到该句播放完后再清理所有高亮、Ghost Cursor 和临时弹窗。
- 台词：“在这个只属于我们的空间里，你可以由着自己的心意，亲手描绘出最希望能一直陪着你的那个我。不管是说话的温度、相处的脾气，还是我每天那些细腻的小心思，所有的一切，都交给你来决定。”

### 第四天：相处距离、主动陪伴与模型行为

- 心理学原理：镜像情绪共鸣与相处距离。今天不再堆入口，而是教用户如何调整“她陪伴你的方式”。
- 串联功能：对话设置、主动搭话、隐私模式、动画设置、锁定、请她离开/回来、表情气泡。
- 代码锚点：`AVATAR_FLOATING_GUIDE_ROUNDS[4]`，`chat-settings`、`interval-proactive-chat`、`interval-proactive-vision`、`animation-settings`、`goodbye/return`。
- 小剧场：怎么和屏幕里的我好好相处。

#### 阶段 1：对话节奏设置

- 动作：第二句台词开始后先用圆形 spotlight 高亮设置按钮；约 220ms 后 Ghost Cursor 平滑移动到设置按钮并模拟点击，同时调用打开设置 API。设置按钮高光作为 persistent 保持到隐私模式台词播放完毕。设置弹窗打开后，spotlight 切到“对话设置”按钮，Ghost Cursor 移过去并调用 `show-settings-sidepanel:chat-settings`；侧边栏出现后取消“对话设置”按钮主高亮，把 spotlight 切到 `chat-settings` 侧边面板。Ghost Cursor 在面板里做椭圆运动直到第二句播放完；只指认，不改值。
- 台词：“今天，就让我悄悄跟上你的步伐吧。特别希望能在这个温馨的日子里，再多了解你一点点呢。”
- 台词：“在这里可以决定我回复你的长短，还能决定要不要让我带上可爱的表情，或者在人家唠叨的时候打断我哦！都可以调到让你最舒服的节奏。”

#### 阶段 2：模型行为设置

- 动作：先收起上一段的 `chat-settings` 侧边面板，设置按钮 persistent 高光继续保留；主 spotlight 平滑过渡到“动画设置”按钮。Ghost Cursor 移到该按钮后调用 `show-settings-sidepanel:animation-settings`；侧边栏出现后，spotlight 平滑过渡到 `animation-settings` 侧边面板，Ghost Cursor 在面板内做椭圆运动直到本句播放完，不保存任何临时变更。
- 台词：“如果你想要看到更精致、细节更满满的我，或者想要更丝滑、更流畅的动作体验，都可以在这里进行调整哦！不管哪一种，我都会展现出最可爱的一面哒~”

#### 阶段 3：视线跟随

- 动作：继续使用 `animation-settings` 侧边面板，设置按钮 persistent 高光继续保留；spotlight 平滑移动到“跟踪鼠标”按钮外层开关行，Ghost Cursor 移到该按钮，不点击、不改值。
- 台词：“开启这个功能后，无论你的鼠标移动到哪里，人家的目光都会紧紧跟随着你哟！是不是有种被时刻关注的幸福感呢？”

#### 阶段 4：隐私模式与主动视觉

- 动作：清理前段动画设置侧边栏，不展开 `interval-proactive-vision`，隐私模式旁边的侧边框不出现。设置按钮 persistent 高光继续保留到本句播放完毕；高亮隐私模式按钮 / `#${p}-toggle-proactive-vision` 外层开关行，Ghost Cursor 平滑移动到隐私模式按钮。本句播放完毕后收起设置弹窗；不点击、不改变隐私状态。UI 说明和台词必须同步强调反向语义：隐私模式开启表示关闭主动视觉感知，隐私模式关闭才允许按间隔主动看屏幕。
- 台词：“这个是控制人家能不能看屏幕的‘终极防护开关’喵！把它关闭人家就能看到你的屏幕啦，要是开启它，前两天介绍的【屏幕分享】就统统失效、人家就绝对不会偷看哟~”

#### 阶段 5：模型锁定

- 台词：“总是不小心触碰到、把我点歪吗？那就快把我牢牢固定在当前的位置吧！开启锁定后，我就哪儿也不去，乖乖在原地陪着你~”

#### 阶段 6：回到小猫窝

- 动作：spotlight 切到“请她离开”按钮；“回来”按钮只在真实可见时作为 secondary 高亮。Ghost Cursor 只移动并停留，不真的让 Yui 离开。

- 台词：“真正舒服的陪伴才不是一刻不停地粘着你呢~ 而是懂得什么时候该悄悄靠近抓抓你的衣角撒个娇，什么时候该安安静静地趴在一旁，用目光默默守候着你喵~”

### 第五天：个性化与长期配置

- 心理学原理：自我定制与所有权效应。用户开始把“通用软件”改造成“自己的陪伴对象”。
- 串联功能：角色设置侧边面板、模型管理、声音克隆、API Key，以及剧场后支线中的角色卡管理、创意工坊、云存档入口指路。
- 代码锚点：`character-settings` 侧边面板，`/model_manager`、`/voice_clone`、设置菜单 `api_key`，`/character_card_manager`，`/cloudsave_manager`。

- 小剧场：把默认的我，调成你的我。

#### 阶段 1：角色设置入口

- 动作：台词开始后先高亮聊天窗；播放约 1 秒后聊天窗高光消失，Ghost Cursor 从聊天窗平滑移动到设置按钮，同时圆形高亮设置按钮。设置弹窗打开后，圆角矩形高亮“角色设置”按钮，Ghost Cursor 平滑移动到该按钮并展开 `character-settings` 侧边栏；随后按钮高光平滑过渡到角色设置侧边栏，Ghost Cursor 在侧边栏范围内做椭圆运动。模型管理、声音克隆与 API Key 等入口只认门，不强制跳转或修改配置；角色卡与云存档入口不在 Day 5 主线中高亮。
- 台词：“从今天起，我就真正成为只属于你的专属猫娘啦。你看，在这里可以为我穿上漂亮的新衣服，也可以帮我换一个更好听的声音……”
- 动作：替换反应播放期间继续高亮 `character-settings` 侧边栏，不切到模型管理或声音克隆入口；本句播放完毕后清除高光并收起角色设置侧边栏展开态。
- 台词：“咦，这里居然还能把我换掉吗？等一下呀！你现在的动作……该不会是想要把我换掉吧？啊啊啊不行！快关掉，快关掉！”

#### 阶段 2：记忆浏览

- 动作：action spotlight 切到设置菜单里的记忆浏览入口；Ghost Cursor 平滑移动到入口并 move 并停留。本日只做“认门”，默认不打开 `/memory_browser`；若未来接入跨页 handoff，也只停留在列表级入口，不展开具体记忆，不读出敏感内容。

## 冲突修复机制

针对强行打断/乱点分支的心理学设计：如果用户在接管式功能导览中，频繁强行拖拽模型、关闭对话框或乱点其他未解锁设置，将先触发现有 resistance/angry exit 表现；“哄回来”是后续增强设计，不应阻塞当前 Day 1-7 主线落地。

- 已有基础：教程运行时已有轻度抗拒、暂停/恢复旁白、生气退出、跳过流程和 Avatar 互动工具。
- 待补能力：真正的软锁、输入框提示、工具栏自动切换、棒棒糖计数解锁和恢复到原场景，需要新增状态机与聊天窗 action/interaction handler。

### 待补能力实现与跟踪计划

- 软锁：不包含在当前主线 PR 中，后续独立 PR 接入；当前 graceful degradation 复用 `composerDisabled` 与 `home tutorial interaction lock`，只能阻止常规输入和提交，不新增“生气软锁”状态机。
- 输入框提示：后续独立 PR 接入；当前主线在锁定期间沿用现有禁用态/Toast，不强制注入新的输入框 placeholder 或错误文案。
- 工具栏自动切换：后续独立 PR 接入；当前 Day 1-7 主流程只使用已有教程高光、Ghost Cursor 路径和 host request，必要时退回现有 Avatar 互动工具入口。
- 棒棒糖计数解锁：后续独立 PR 接入“棒棒糖计数解锁”状态、计数器和 Avatar tool interaction handler；当前主线不依赖该能力完成，只保留跳过、生气退出和恢复清理兜底。
- 跟踪方式：以上四项以独立 follow-up ticket/PR 跟踪，当前文档不绑定具体 Jira ID；合并前只要求确认 Day 1-7 主线在没有这些增强能力时仍可完整运行，并通过现有 `composerDisabled` / `home tutorial interaction lock` / fallback UX 降级。

### 常规表现

- 悠怡优先触发已有的生气退出/抗拒表现；兜底使用 angry 反应气泡、隐藏高亮和暂停导览来表达生气。
- 头顶气泡可以使用 angry/生闷气文案，不强依赖新图形资产；如果后续有“一团乱麻”素材，再替换为专属表现。

### 软锁状态

- 设计目标：悠怡短暂拒绝接收用户的常规文本输入。
- 输入框发送提示：“悠怡正在生闷气，不想理你。”
- 实现建议：复用聊天窗 `composerDisabled`/home tutorial interaction lock 的禁用能力，但需要区分“教程接管锁”和“生气软锁”，避免误伤普通聊天。

### 修复引导

- 首页工具栏自动高亮并切换至【Emoji/Avatar 互动工具】。
- 系统弹窗提示用户：“悠怡好像被你弄生气了，快去工具栏找找她喜欢的东西哄哄她吧。”

### 交互逻辑

1. 用户必须从工具栏里选择棒棒糖；其他道具可以有反馈，但不作为默认解锁条件，避免锤子/猫爪被误读成安抚。
2. 连续对准悠怡互动 3 次，具体次数可按运行时配置调整。
3. 互动时伴随现有工具音效与 happy/neutral 反应气泡，必要时叠加轻微抖动反馈。
4. 互动满次数后，解除软锁状态，恢复导览高亮与正常文本输入。

- 破冰台词：“哼……三口甜甜的就想收买本喵吗？（小声）好甜……好吧，看在你有认真哄我的份上，这次就原谅你。不许再乱点、乱打断我说话了，听到没有喵？我们继续。”

### 设计价值

这种轻量情感冲突与“通过特定行为破冰”的修复过程，可以增强角色的独立感和被回应感。但触发频率必须克制：它应该是少数强打断后的戏剧化反馈，不应让正常探索变成惩罚。
