# 记忆分析导出使用说明

本页面向**测试人员 / 分析相关人员**, 讲清「一键脱敏导出」怎么用、导出包里有什么、脱敏挡位分别做什么、以及有哪些局限。

导出功能的目标: **一键去除隐私敏感数据, 得到一个干净、可共享、可离线分析的记忆数据包**, 里面既有脱敏后的**原始记忆**, 也有我们已经分析出的**二级结论**。

---

## 1. 在哪里、怎么用

1. 打开顶部 **「记忆系统分析」** workspace, 停在默认的 **「系统概况」** 子页。
2. 选好角色、等概况分析跑完 (页面进入 ready 状态)。
3. 工具栏右侧出现 **[导出记忆分析]** 按钮, 点击弹出导出小窗。
4. 在小窗里:
   - 选 **脱敏挡位** (默认 `标准 standard`, 见 §3)。
   - 勾选是否 **包含对话语料** (`conversation_corpus`, 默认包含; 见 §4)。
   - 窗口内固定显示一块**黄色高亮「记忆脱敏说明」** (务必阅读, 不折叠), 就地讲清挡位解释与局限。
5. 点 **[导出]**, 浏览器弹出**「另存为」窗口**让你选择保存位置 (Chromium 系浏览器; Firefox/Safari 则按浏览器默认下载目录直接下载)。取消「另存为」不会报错, 可重新导出。

> **文件名**: 形如 `NEKO testbench_记忆导出_<角色标识>_<日期>.zip`。为与脱敏一致, `<角色标识>` 在 `minimal` 挡用真实角色名, 在 `standard` / `strict` 挡用中性占位「角色」——**文件名本身不泄漏**已在包内假名化的身份。

导出是**纯读操作**: 不会修改、不会保存、不会给这个会话留下任何副作用 (不取会话锁、不触发自动保存)。也**不调用任何大模型**, 离线、零成本。

---

## 2. 导出包 (ZIP) 里有什么

```text
NEKO testbench_记忆导出_<角色标识>_<日期>.zip
├── README.md                  人读说明 + 本次用的脱敏挡位解释
├── manifest.json              清单: 挡位 / schema 版本 / 文件描述 (不含反查表)
├── raw_data/                  脱敏后的原始记忆
│   ├── recent.json            近期对话窗口
│   ├── facts.json             抽取出的事实
│   ├── reflections.json       反思
│   ├── persona.json           人设
│   └── conversation_corpus.json   对话语料 (仅当勾选"包含对话语料")
└── analysis/                  机器可读的二级分析结论
    ├── summary.md             系统概况摘要 (人读)
    ├── overview.json          系统概况 (需关注项 / 卡片 / 全部发现)
    ├── lineage.json           记忆溯源快照
    └── embedding_*.json       向量空间: 体检覆盖率 / 近重复 / 聚类 / 语义源vs结构源
```

- `raw_data/` = **原始记忆** (脱敏后)。
- `analysis/` = 基于这些记忆分析得到的**二级结论**, 主体为机器可读 JSON (方便再加工), 另含一份人读的 Markdown 摘要 `summary.md`。
- 所有 JSON 均为 UTF-8、缩进格式化, 直接可读可解析。
- 缺失的数据 (例如角色还没跑过向量嵌入) 会以**警告**形式记录, 不会让导出失败——你会得到一个"尽力而为"的包。

## 3. 三档脱敏 (默认 standard)

| 挡位 | 做什么 | 适用场景 |
|---|---|---|
| `minimal` 最小 | 只移除**凭据类**敏感值 (api_key / token / cookie 等), 其余原样保留 (含真实姓名与全部正文) | 内部信任环境、只想去掉密钥 |
| `standard` 标准 (默认) | 在 minimal 基础上, 把**身份名** (主人名 / 角色名) 统一替换为中性占位符 (如 `<角色>` / `<用户>`), 其余正文保留 | 大多数共享场景 |
| `strict` 严格 | 在 standard 基础上, **整层撤下原始逐轮转录** (recent 消息正文 / conversation_corpus 正文 → `<omitted len=N role=...>` 占位), 但**保留**已假名化的派生记忆 (facts / reflections) | 对外分享、只要分析价值不要原话 |

### 关键保证: 跨层一致性

无论哪一挡, **凡是参与假名化映射的身份名** (即长度 ≥ 2、未被 §5「过短身份名」规则跳过的名字), 在**对话正文**与**事实 / 反思 / 人设 / 分析结论**里都被**同一套映射**替换。绝不会出现"对话里叫 A、事实记忆里却叫 B"这种自相矛盾的情况。占位符替换在整包**统一进行一次**, 覆盖所有嵌入文本 (包括分析结论里的引用文本)。(过短身份名两侧都不替换、原样保留, 因此同样不会错位——但它不在"已脱敏"范围内, 见 §5。)

`strict` 挡撤转录时也遵守这一点: 撤的是**原始逐轮转录整层**, 保留的是**派生记忆**——因此不会制造"转录说一套、事实说另一套"的错位。

## 4. 是否包含对话语料

`conversation_corpus.json` 是完整对话语料 (来自 `time_indexed.db`), 体量可能较大。

- **勾选 (默认)**: 导出包含 `raw_data/conversation_corpus.json`。在 `strict` 挡下其正文会被撤为占位符。
- **取消**: 导出**完全不含**该文件, 包更小、隐私面更窄。

其余原始记忆 (recent / facts / reflections / persona) 不受此开关影响, 始终包含。

---

## 5. 局限 (务必知晓)

- **自由文本里的隐私不会被自动清洗**。脱敏只处理"凭据"与"身份名"两类**结构化/可枚举**的标识。如果用户在对话或事实正文里**自己写进了**地址、电话、身份证号等隐私, 这些**不会**被任何挡位自动识别和删除。若要对外分享此类数据, 请选 `strict` (撤原始转录) 并**人工复核所有含自由文本的层**: 至少包括 `raw_data/facts.json`、`raw_data/reflections.json`、`raw_data/persona.json` (人设简介 / 喜好等自由字段), 以及 `analysis/` 下任何含正文或引用文本的文件 (如 `lineage.json` 的非转录字段、聚类标签、AI 报告)。
  > 这是刻意的设计: 本工具不做基于正则/启发式的"内容猜测式过滤", 以免误删或漏删给你**虚假的安全感**。
- **导出包不含反查表**: `manifest.json` 只记录用了哪个挡位, **不**包含"占位符 → 真实名"的映射。拿到包的人无法据此还原身份。
- **过短的身份名不参与替换 (会原样保留, 需人工处理)**: 为避免把常见短词误当成名字一并替换掉 (over-redaction), **少于 2 个字符**的身份名 (主人名 / 角色名) 会被跳过、**原样保留在正文与事实/反思/分析结论里**。此时 `manifest.json` 的 `warnings` 会显式记一条提醒。**这意味着即使选了 `standard` / `strict`, 这类过短身份名仍会出现在可分享包中** —— 若涉及真实身份, 请在对外分享前**人工替换或复核**。

## 6. 常见问题

- **Q: 导出会不会改动/污染我的会话?**
  不会。纯读、不写盘、不取锁、不 autosave、不调模型。

- **Q: 为什么某些 `analysis/embedding_*.json` 是空的或带警告?**
  说明该角色还没有可用的向量嵌入 (或数据不足)。导出会如实记录警告而非报错。

- **Q: 我要对外公开分享, 选哪挡?**
  选 `strict`, 并人工过一遍所有含自由文本的层: `raw_data/facts.json`、`reflections.json`、`persona.json`, 以及 `analysis/` 下含正文/引用的文件 (见 §5 局限)。脱敏只处理凭据与身份名, 自由文本里的隐私需你自己确认。

- **Q: 想比较不同时间点的记忆怎么办?**
  分别导出两个包, 用 `manifest.json` 里的时间戳与 schema 版本对齐后再 diff。
