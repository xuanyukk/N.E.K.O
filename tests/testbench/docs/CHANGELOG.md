## 版本号约定

- **MAJOR** bumps (X.y.z) — 某个 Phase 签收, 且带来**外部可见契约变更**
  (沙盒目录结构 / 导出格式 / 持久化 schema / HTTP 端点重命名等).
- **MINOR** bumps (x.Y.z) — 新增特性, 但不破坏现有测试者的使用流程
  (新面板, 新按钮, 新 Op, 新诊断页面).

## v1.12.1 (2026-07-15) — 导出健壮性修复 (PR AI 审核跟进)

针对 PR #2363 的 AI 审核意见做的一批**纯修复/打磨**, 不新增功能, 不改变使用流程。

### 修复

- **strict 记忆导出不再经"分析层"泄漏原始对话**: 此前 `strict` 档虽已撤掉 `raw_data/` 里的对话正文, 但 `analysis/lineage.json` 的会话节点仍保留逐字对话 (`label` / `meta.content`)。现已一并撤除, `strict` 导出对外分享时不会再从分析层漏出被撤掉的对话 (与"整层撤原始转录"保持跨层一致)。
- **角色一键导出遇到读不出的记忆文件时改为明确报错, 不再静默产出残缺备份**: 目录内某个文件读取失败时, 导出会以 500 `ExportReadFailed` 中止, 而不是跳过它后仍返回一个"看似完整"的 200 备份包。
- **角色导出加符号链接逃逸防护**: 会跳过符号链接以及真实路径落在角色记忆目录之外的条目, 避免 `rglob` 顺着链接把无关文件打进这个**不脱敏**的备份包。**(二轮跟进)** 进一步堵住"角色记忆目录**本身**是符号链接"的根目录逃逸——此时 `memory_dir/<name>` 若解析到记忆根目录之外, 直接以 500 `UnsafeCharacterDir` 中止导出 (仅当整个 `memory_dir` 被整体迁移/软链时才放行)。
- **过短身份名的保留行为显著披露 (二轮跟进)**: 少于 2 字符的身份名会被跳过、原样保留在 `standard`/`strict` 包中 (避免误删常见短词), 现已在导出说明与用户手册**显著标注**其仍会出现在可分享包内、涉及真实身份需人工处理。
- **超大文件先按体积拦截再读取**: 角色导出对单个大文件 (如 `time_indexed.db`) 先按磁盘体积判上限, 越限即 413, 不再先整块读入内存后才拦截。
- **下载文件名响应头对双引号更稳健**: 记忆导出 / 角色导出的 `Content-Disposition` ASCII 兜底名里若混入双引号, 不会再截断响应头。
- **minimal 档导出说明文案更正**: `minimal` 档实际**保留真实身份名** (仅去密钥), 说明文案不再错称其"仅假名化身份"。

### 内部 (对使用者无感, 详见 PROGRESS)

- 记忆导出 / 角色导出前端共用一份 `core/download.js` (文件名解析 + 另存为/anchor 兜底), 消除两处重复实现的分叉风险; 身份假名化替换改为单遍正则交替, 杜绝占位符被二次改写; 补 p41/p42/p43 相关断言与用例; 修正若干文档与 docstring 与实现对齐。

## v1.12.0 (2026-07-15) — 代码线索 (开发者·反推)

在「记忆系统分析」工作区新增第 4 个子页「**代码线索 (开发者)**」。它把系统概况里**机械不变量类**的分析发现, 反推成"值得排查主程序记忆代码哪个模块/写入路径"的**排查线索**, 仅供代码相关人员参考——**普通测试员可忽略本页**。**新增特性, 不改变现有使用流程。**

> **务必先读**: 这些是**排查方向 / 线索, 不是 bug 报告**。由记忆数据反推代码问题**不完全可靠**, 可能是假阳性 (数据/模型/配置原因, 未必是代码)。每条都需你到主程序代码里**人工确认**后才能下结论。本页**只读**, 绝不会自动修改任何代码。

### 新增

- **代码线索子页 (记忆系统分析 → 代码线索 (开发者))**: 一键列出由记忆分析确定性反推出的排查线索。每条线索给出: 违反了哪条机械不变量、证据强度 (较强/中等/较弱)、疑似指向的主程序模块 (方向, 非精确定位)、还需哪些运行期证据才能坐实、以及一句祈使式排查建议。
- 覆盖的机械检查: 悬空来源引用、无溯源边的晋升人设、向量过期/损坏/多空间、游离反思, 外加两个新增的确定性检查——**同文件主键重复 (ID-DUP)** 与 **events.ndjson 重复 event_id (EVT-DUP)**。
- **刻意排除内容质量类**: 矛盾 / 冗余 / 归因 / 晋升漂移 / 留存等问题**不做代码反推** (它们属数据/模型/配置层), 页面单独列出这些类别的计数并说明"请走记忆运营而非改代码"。

### UI (本次重点: 防误导)

- 页首**固定、不可折叠、不可关闭的红色警告框**, 逐条讲清"这是方向不是结论、不完全可靠、需人工确认、只读不改代码、内容质量类不在此反推", 末条是一个可点开的**说明文档链接**。
- **措辞纪律**: 线索卡不使用"bug/缺陷/确认存在"等确定性断言, 一律"疑似/建议排查/仍需人工确认"; 强度可视化**克制** (即便较强也不用大红块)。
- **诚实空态与状态**: 没有线索时明确提示"**不代表代码一定没问题**"; 向量检查/事件流检查会区分"已运行"与"未检查 (不代表通过)"。
- 新增面向使用者的说明文档 **[代码线索使用说明](code_leads_guide.md)** (浏览器 `/docs/code_leads_guide`): 讲清怎么读一条线索、检查了哪些机械不变量、为什么排除内容质量类、以及局限。页首警告里的链接直达此文档。

### 隐私 / 安全

- 纯读当前会话沙箱内的记忆数据, 不调用任何 LLM, 不写任何盘, 不改会话状态。线索样例仅含记忆条目 id/标签, 仅供本地查看, 请勿截图外传。

## v1.11.0 (2026-07-15) — 角色一键导出 (备份/迁移)

在「Setup → 导入 (Import)」页「从真实角色导入」区的**每个本地角色**行, 于 [导入] 旁新增 [导出] 按钮, 与"从本地导入"构成一对**镜像操作**。一键把该角色在主程序里的**完整记忆目录**忠实打成 `<角色名>.zip`, 用于**本地备份 / 迁移到另一台机器 / 分享给可信的相关人员**。**新增特性, 不改变现有使用流程。**

### 新增

- **角色一键导出 (导入页每个本地角色行)**: 点 [导出] 即得 `<角色名>.zip`。内部结构与主程序记忆目录一致——顶层是以角色名命名的文件夹, 内含 `characters.json` + 该角色的全部记忆文件 (`persona.json` / `facts.json` / `reflections.json` / `recent.json` / `time_indexed.db` / `events.ndjson` / `outbox.ndjson` / `reflection_archive/` 等)。
- **选择保存位置**: 导出时 (Chromium 系浏览器) 弹出**「另存为」窗口**让你选保存位置, 默认文件名 `<角色名>.zip`; 其它浏览器回退为直接下载。
- **导出/导入闭环**: 导出的 zip 可原样用「导入 → 从 zip 人格档案导入」再吃回沙箱, 无需手动改结构。

### 修复 / 加固

- **打包去重**: 交付后以样例做「原样导入再导出」自测发现——当角色记忆目录里**也**存在一份 `characters.json` (例如导入"配置+记忆平铺在一个文件夹"的档案后) 时, 导出包会出现**重名成员**导致产物畸形。已修: 打包时对成员去重, 配置目录的 `characters.json` 优先。自测同时确认**导出本身无损** (把角色记忆目录直接打包 = 与原始逐字节一致)。

### 隐私 / 安全

- **此导出完全不脱敏 (刻意设计)**: 与「记忆分析 → 一键脱敏导出」(v1.10.0) 目的相反——那个是给外部分享分析结论、会脱敏; **这个是忠实全量转储**, 保留真实姓名 / 完整对话 / 事实 / 向量库, 供备份与可信迁移。按钮 tooltip 已明示"含隐私原始数据, 仅供本地备份 / 迁移 / 可信分享"。请勿把此导出包公开分享。
- `characters.json` 是**全量**配置 (含主人名与其它角色), 与主程序原文件一致。
- 纯读宿主文件系统, 绝不写主程序目录, 不改会话状态。

## v1.10.0 (2026-07-15) — 记忆分析: 一键脱敏导出

在「记忆系统分析 → 系统概况」页新增 **[导出记忆分析]** 按钮, 一键把当前角色的**原始记忆**与**记忆分析结论**打成一个 ZIP 包, 用于**共享/交接/离线分析**。**新增特性, 不改变现有使用流程。**

### 新增

- **一键脱敏导出 (系统概况页工具栏)**: 概况分析完成 (ready) 后, 工具栏右侧出现 [导出记忆分析]。点击弹出小模态, 选择脱敏挡位与是否含对话语料, 点 [导出] 即下载一个 ZIP。ZIP 内分两个文件夹:
  - `raw_data/` — 脱敏后的原始记忆 (recent 近期对话 / facts 事实 / reflections 反思 / persona 人设 / 可选 conversation_corpus 对话语料)。
  - `analysis/` — 机器可读的二级分析结论 (系统概况 overview / 记忆溯源 lineage / 向量空间体检·近重复·聚类·语义源vs结构源)。
  - 另含根 `README.md` (人读说明 + 脱敏挡位解释) / `manifest.json` (清单 + 挡位 + schema 版本) / `analysis/summary.md` (概况摘要)。
- **三档脱敏 (默认 standard)**:
  - `minimal` — 只去除凭据类敏感值 (API key / cookie / token 等), 其余原样。
  - `standard` (默认) — 在 minimal 基础上, 把角色名 / 主人名等**身份名**统一替换为中性占位符 (如 `<角色>` / `<用户>`)。
  - `strict` — 在 standard 基础上, **整层撤下原始对话转录** (recent 消息正文 / conversation_corpus 正文替换为结构占位符 `<omitted len=N role=...>`), 但**保留**已经过假名化的派生记忆 (facts / reflections), 以保住分析价值。
- **跨层一致性保证**: 同一个身份名在"对话正文"与"事实/反思/人设/分析结论"里被**同一套映射**替换, 不会出现"对话里叫 A、事实里却叫 B"的错位。
- **UI 脱敏说明**: 模态内含**固定显示的黄色高亮**「记忆脱敏说明」(务必阅读, 不折叠), 逐条讲清三挡分别做什么、一致性保证、以及**局限**——自由文本里用户自己写进正文的隐私 (地址/电话等) 不会被自动清洗, 请按需选 strict 或人工复核。
- **友好文件名 + 选择保存位置**: 导出包命名为 `NEKO testbench_记忆导出_<角色标识>_<日期>.zip` (不再是早期的编号式命名)。`<角色标识>` 与脱敏挡位一致——`minimal` 用真实角色名, `standard`/`strict` 用中性占位「角色」, 文件名本身不泄漏身份。导出时 (Chromium 系浏览器) 弹出**「另存为」窗口**让你选保存位置。

### 隐私 / 安全

- **纯读、不落任何盘写**: 导出端点是只读的, 不取会话锁、不触发 autosave, 与 `/overview`、`/lineage` 一致, 不会给被导出的会话留下副作用。
- **不反向泄露映射**: `manifest.json` 只记录用了哪个挡位, **不**包含"占位符 → 真实名"的反查表。
- **零 LLM / 零联网 / 零成本**: 导出只消费磁盘已有数据与非 LLM 聚合器, 不调用任何模型, 离线可跑。

### 说明文档

- 新增 tester 可见文档 [memory_export_guide](memory_export_guide.md) (记忆导出使用说明), 已接入 Settings → About 的 `/docs` 深链。

## v1.9.4 (2026-07-01) — PR 代码审核意见处理 (安全/健壮性加固, 纯修复)

PR #2118 上 CodeRabbit / Codex / Greptile 三个 AI 审核器提出的意见, 逐条核实后**修复了确属真实的问题**, 跳过了设计本意/误报项。**纯修复与健壮性加固, 无新功能, 不改变使用流程。**

### 安全 / 健壮性

- **zip 人设导入的"解压炸弹"防护改为按真实字节计量**: 之前 500 MiB 上限是累加压缩包元数据里的 `file_size` 字段 (可被伪造为 0) 来判断, 且导入前先 `testzip()` 把每个成员全量解压一遍。现在改为**边解压边累计真实写出字节**, 一旦超限立即中止; 去掉了会被当作"炸弹放大器"的预扫描 (损坏仍能在流式拷贝时由 CRC 校验发现)。
- **角色名作为目录名前先做路径安全校验**: 压缩包里的角色名若含 `/`、`\`、`..` 等路径字符, 现在会被拒绝 (422 `UnsafeCharacterName`), 不会再被直接用于拼接沙盒写入路径。(注意: 这是对"文件系统标识符"的结构校验, **不是**对用户内容的过滤——facts/人设正文等内容一律不过滤。)
- **角色 memory 目录回退查找用字面匹配**: 之前 `rglob(角色名)` 把角色名当 glob 模式, 名字里若含 `*?[]` 会误匹配到别的目录; 改为枚举后按名字字面比较。
- **导入时 facts.json 改为原子写**: 复用全项目统一的 `atomic_write_json` (tmp + fsync + 原子替换), 写一半被强杀不会留下半截文件。

### 修复

- **归因 LLM 精判回复无法解析时不再"静默成功"**: 之前若模型返回的不是 JSON 数组 (或空回复), 会被当成"没有来源"静默返回 `method="llm"` 且零条边; 现在会**显式回退**到文本相似度并透出 `llm_fallback` 原因 (延续 v1.9.3 "回退不静默" 原则)。模型若正常返回 `[]` (确无来源) 仍按诚实的 LLM 结果保留。
- **重复的 recent.json 对话不再被合并丢失**: 之前同 role 同内容的多条对话 (如多个"好的") 会因 id 相同被溯源图去重塌缩成一条; 现在 id 计入条目序号, 重复对话各自成节点。
- **修正 P29 蓝图里的一处乱码** ("晋升漏斗比率")。

### 性能

- **概况/向量空间的耗时计算改为卸载到线程池**: `attribute_all` 文本归因、簇标签的聚类计算 (HDBSCAN/numpy) 不再在事件循环线程里同步跑, 避免卡住其它请求。
- **归因/溯源读取对话语料时套用行上限**: 不再用 `limit_rows=None` 把整张 `time_indexed.db` 读进内存, 而是套用扫描上限 / 内置 5000 行默认值。
- **向量坐标缓存加锁**: 多请求并发 (线程池) 下对坐标缓存的 check-then-set + FIFO 淘汰加 `threading.Lock`, 避免重复降维与字典竞态。

### UI 健壮性

- **多处异步面板加"过期响应"守卫**: 系统概况 / 记忆溯源 / 向量空间在切换会话或快速点击时, 上一请求的慢响应不会再覆盖新页面状态; 向量散点/矩阵的 `ResizeObserver` 在画布重建后会重新绑定。
- **修正事实节点左侧色条** (`stroke-left` 非法属性 → `stroke`)。

### 未采纳 (设计本意 / 误报)

- "系统概况「引用了已删除的事实」下钻到向量空间" 维持现状: 向量空间的「语义源 vs 结构源」是**唯一**会显式标出"已删除/缺失来源"的视图, 是该发现最相关的落点。
- "umap 在线安装端点需登录" 维持现状: 它是**环境级能力开关**而非角色相关操作, 本地单用户工具且按钮只在向量空间页 (本就有会话) 出现, 已在 docstring 注明刻意如此。
- CodeRabbit 的若干 markdown 格式 nitpick (代码块语言标签 / 表格空行) 为内部蓝图文档的低价值格式建议, 不影响 CI, 暂不处理。

## v1.9.3 (2026-07-01) — LLM 失败回退不再静默

### 改进

- **记忆溯源「分析来源 (LLM 精判)」失败回退会明确告知**: 之前点 LLM 精判推测来源时, 若模型不可用/失败, 会**静默回退**到文本相似度——你看到的虚线其实来自文本相似度而非 LLM, 却没有明显提示 (只有一闪而过的 toast)。现在详情栏里会**持久显示**一条提示: "LLM 精判失败, 已回退文本相似度归因到 N 条对话……原因: ……", 让你清楚这些虚线的来源与失败原因概括。
- **其它涉及 LLM 失败回退的机制本就已透出原因, 本次确认对齐**: 向量空间的「簇标签 LLM 概括」失败会回退到每簇代表记忆 (medoid) 并在侧栏持久写明原因; 系统概况的「AI 体检报告」「矛盾 NLI 裁决」失败会显示"不可用"并列出原因 (例如该填哪个 API)。即"LLM 失败回退一律不静默"。

## v1.9.2 (2026-07-01) — 系统概况: 新增发现「引用了已删除的事实」

### 新增

- **「引用了已删除的事实」(系统概况 → 结构)**: 当一条反思声明的来源事实 (`source_fact_ids`) 指向**已被硬删除**的事实 (注意: 被"吸收 absorbed"的事实仍然在册, 不算) 时, 概况会明确列出"有 N 条反思共引用了 M 处已不存在的事实"。**已删除的事实不应再被后续节点引用**——这通常是删除事实时没有同步清理引用它的反思所致, 会让这些反思的溯源链出现断点。一键下钻到「向量空间 → 语义源 vs 结构源」即可看到具体是哪条事实 (标 `(已删除/缺失)`)。
  - 它与既有的「无来源的反思」(D1) 不同: D1 是"一条有效来源都没有", 而本发现也能抓出"**只有部分**来源被删、其余仍在"的情况。引用越多, 严重度越高 (≥10 处标为"严重")。

## v1.9.1 (2026-06-30) — 系统概况/向量空间: 两处发现展示与判定修复

测试中发现并修复了两个会误导判断的问题, **纯修复, 无新功能**:

### 修复

- **「无来源的人设」误报 (系统概况 → 结构孤儿)**: 之前对"由多条反思**合并晋升**而来的人设", 因为它那唯一的 `source_id` 在合并过程中已被消解 (不再是独立反思节点), 会被错误地报成"来源反思不存在"——可在「记忆溯源」图里却明明能看到它的来源 (那些被合并的反思)。现在该判定改为**直接复用溯源图自己画出的来源边** (`promoted_from` / `merged_from`), 与你在溯源图里看到的完全一致: 只有**真正一条来源边都没有**的人设才会被列为"无来源"。
- **「声明却不相近」里只显示一串序号 (向量空间 → 语义源 vs 结构源)**: 之前当一条反思声明的来源事实**没有向量** (未嵌入) 时, 该条目只显示形如 `fact_20260613181531_3d2bdf0e (∅)` 的裸 id, 没有任何可读信息。现在改为显示该事实的**真实文本内容**, 并以 `(未嵌入, 无法比对语义)` 标注原因 (若该事实确已删除则标 `(已删除/缺失)`)。

## v1.9.0 (2026-06-30) — 记忆分析系统: 新增「系统概况」入口子页 (自动排查 + 体检报告)

给 **记忆系统分析** 工作区加了第三个、也是**默认入口**的子页 **「系统概况」**。它不读原始数据, 而是把已有的「记忆溯源」(P27) 与「向量空间」(P28) 两个分析**聚合成一屏图景**, 自动排查记忆系统当前的问题, 每条发现都能一键下钻到对应子页看详情。**纯只读, 不会修改任何记忆。**

### 新增功能

- **一屏概览卡片** — 记忆构成 / 嵌入覆盖 / 向量空间 / 聚类 / 流水线漏斗 / **结论可信度**, 一眼看清规模与健康度。
- **「N 项需关注」+ 自动发现清单** — 不给虚的"健康分", 而是逐条列出**具体问题**, 带严重度 (严重/注意/提示)、所属**功能环节** (抽取/去重/反思/晋升/纠正/嵌入/结构) 与一键下钻按钮。覆盖:
  - **冗余重复**: 近重复对、重复簇、可合并的冗余代价。
  - **矛盾记忆 (诚实分层)**: 已落盘的真矛盾 (纠正记录 / 被否决反思 / 被抑制人设)、**未解决的矛盾** (纠正了但旧文还在)、以及"**同主题待核对候选**"——后者只是"值得复查", **相似 ≠ 矛盾**, 不会武断判定。
  - **归因偏离 / 结构孤儿 / 嵌入健康 (缺失·过期·损坏·维度分裂)**。
  - **流水线吞吐**: 晋升停滞、反思积压、否决率偏高。
  - **晋升保真度** (人设与其来源反思是否语义漂移) 与 **留存质量** (高价值事实被冷落 / 低质量事实 / 僵尸事实)。
- **结论可信度 (元诊断)** — 明确告诉你"这次结论基于哪些数据、缺了什么" (如无向量 / 覆盖偏低 / 缺对话时间库), 不当黑箱。
- **可选 LLM 能力**:
  - **LLM 体检报告** — 让记忆模型基于上面的只读统计给出总体判断与优先建议 (仅建议, 不改记忆)。
  - **矛盾 NLI 裁决** — 对"待核对候选"逐对做自然语言推理, 判断是真矛盾 / 重复 / 互补 / 无关。
  - 两者在模型不可用 / 失败时都会**优雅降级并告知原因** (例如该去填哪个 API), 不会报错。

### 说明

- 「系统概况」现在是进入"记忆系统分析"后的**默认子页**; 仍可在左侧子页菜单切到「记忆溯源」「向量空间」。
- 向量相关的排查 (冗余 / 归因 / 晋升保真 / 矛盾候选) 需要角色已有向量; 没有向量时这些项会标注不可用, 但结构 / 流水线 / 已记录矛盾等排查仍照常进行。

## v1.8.1 (2026-06-30) — 修复: 向量空间加载会卡住其它页面 + LLM 概括失败给出原因

### 修复

- **向量空间加载不再冻结整个界面**: 之前进入向量空间触发降维 (PCA/UMAP) 等较重计算时, 这些计算是在服务主线程里**同步**跑的, 会把服务器**卡住几秒**, 期间你切到其它子页 / 其它功能大页都会一直显示"加载中"。现在这些计算 (散点 / 最近邻 / 语义源 / 近重复 / 相似度矩阵) 都移到后台线程, 加载向量空间时**不再阻塞其它页面**。
- **LLM 概括聚类失败时告诉你原因**: 点「用 LLM 概括聚类」若失败 (最常见是**没配记忆模型 / API**), 现在右栏会**直接显示具体原因**并弹提示, 例如"请先在 Settings → Models → memory 填好 base_url 与 model。"——明确告诉你该去填哪个 API, 而不是只说一句"已回退"。

## v1.7.0 (2026-06-30) — 向量空间: 近重复 + 相似度矩阵

把**向量空间**子页剩下的两个分析视图补齐。现在散点 / 近重复 / 相似度矩阵 / 语义源 vs 结构源
四个模式都可在工具栏切换。

### 新增功能

- **近重复 (④)** — 新模式, 找出向量上**高度相似 (cosine ≥ 阈值)** 的记忆对 (跨类型也算, 例如
  一条事实和一条反思说的几乎是同一件事)。右栏有**相似度阈值滑块** (0.80–0.99): 拖动即重算;
  超阈值的对会在散点图上**用红线连起来** (越相似越深), 右栏列出每一对 (分数 + 两端摘要),
  点列表里的一对会在图上高亮。对数过多时只列分数最高的若干对。
- **相似度矩阵 (⑤)** — 新模式, 对一个**子集** (默认取当前类型筛选下的记忆, 上限 80 条) 画
  **NxN cosine 热力图** (canvas 绘制): 行列已**按相似度聚类重排**, 因此相近的记忆挨在一起,
  成块的深色区即一簇近义记忆。颜色越深 = 越相似, 对角线恒为 1; 悬停任意格子看是哪两条记忆
  及其分数。子集超过上限会提示并截断。

> 这两个视图都是**只读**的, 与既有的散点 / 语义源视图共享同一套覆盖率体检与类型筛选。

在**向量空间**子页的散点视图里新增**降维算法切换**: 默认仍是 PCA (零依赖、永远可用),

    选中/聚焦只用节点卡片本身的描边表示.
- **Memory Trace 大图不再卡死 + 虚线按需显示** (手测反馈: 点「推测全部源头」后,
  上百条对话连到众多记忆, 连线糊成一团、画面卡顿到无法拖动):
  - **虚线连线分层显示 (LOD)**: 总览只画**结构实线** (真因果); 启发式**虚线**仅在数量
    不多时才在总览整体画出, 一旦超过阈值就**默认隐藏**, 左下角提示隐藏了多少条 ——
    **点选任意记忆节点即只显示它这一条链路的来源虚线**, 既看得清又不卡. 聚焦某节点时
    也只渲染**该子树内的连线**, 不再把全图连线都画出来 (再淡化).
  - **去掉海量虚线的箭头标记**: 箭头 marker 是 SVG 大规模渲染的主要性能瓶颈, 现在只有
    (数量很少的) 结构实线带箭头; 虚线方向由左→右泳道布局表达. 连线层不参与鼠标命中、
    并开启 `optimizeSpeed`, 拖拽/缩放明显跟手.
  - **「推测全部源头」自带上限**: 每条记忆最多取 3 条最相似来源, 且全批连线有总量上限,
    超出会截断并提示 (想看某条记忆的完整来源, 选中它再点「分析来源」). 海量对话的角色
    不会再一次性返回上千条连线把前端压垮.
- **Memory Trace 聚焦交互精简 + 聚焦时自动紧凑重排** (手测反馈):
  - **去掉多余按钮, 只留一行提示**: 删掉右上角「总览 / 聚焦」切换按钮和节点详情里的
    「聚焦此节点 / 退出聚焦」按钮 —— 现在**点选节点即聚焦、点击空白即取消**, 更快捷; 图上方
    用一行文字说明这一交互.
  - **聚焦 = 自动重排到最小画面**: 聚焦某节点时, 图会**只保留该记忆的整条溯源链** (本身 +
    全部上游 + 全部下游) 并**重新紧凑排布**, 把这些节点排到最近的距离、缩放到恰好完整呈现;
    无关节点直接移出画面 (不再只是淡化). **取消聚焦后自动恢复原来的整图排布**.
    重排同样用**森林布局, 并把被聚焦的节点 (根) 放在整棵子树的纵向正中** —— 它的上游/下游
    分支对称地分布在它两侧, 而不是被挤到顶端.
- **Memory Trace 下游节点(反思/人设)现在竖直居中在它的来源上** (手测反馈): 之前在总览里, 一条
  反思常常飘在它自己那一组事实的**上方**, 而不是居中. 原因是排版分两步、各泳道**各自独立对齐**,
  下游节点用的是上游节点"对齐前"的旧位置, 上游一移位下游就对不上了. 现在改为**从最上游(对话)向
  下游(事实→反思→人设)逐层对齐, 每个下游节点都按它"已经排好位"的来源重新取中点** —— 于是每个
  反思/人设都精确落在它来源们的纵向正中, 不再飘到块外.
- **Memory Trace 「推测全部源头」结果现在真的显示出来 + 聚焦显示完整对话子树** (手测反馈):
  - **总览显示归因结果**: 之前点「推测全部源头」后, 对话/摘要来源节点在总览里几乎总被
    "按需隐藏"逻辑挡掉, 看不到结果. 现在归因出的来源节点会**直接出现在总览**并用虚线连到
    对应记忆 (后端对批量归因有硬上限: 每条记忆前 3 条、全批最多 400 条连线, 所以总览能完整
    画出). 性能靠"虚线不带箭头 + 连线层不吃鼠标命中"保证, 不再靠隐藏连线; 仅当反复手动归因
    把虚线累积到远超上限的病态情况才会临时隐藏并提示.
  - **聚焦显示整条链路 (含对话子树)**: 之前点一条反思只显示它的源事实, 看不到事实下面的对话.
    现在聚焦任意记忆都会**沿真因果 + 已归因来源完整多跳展开** —— 聚焦反思 = 反思 ← 事实 ←
    对话整条溯源链一起显示, 聚焦人设也会一路展开到底层对话; 子树仍用森林布局紧凑重排、焦点居中,
    深链路也清晰.
  - (此前一版曾为治"事实被隐藏虚线拉散"而把隐藏虚线移出布局并把聚焦的猜测来源限制成一跳,
    导致上面两个问题, 现已修正.)
- **Memory Trace 节点显示对话原文, 不再是 JSON** — 对话节点此前把结构化消息内容
  (`[{"type":"text","text":"..."}]`) 原样塞进卡片; 现在抽取并拼接文本, 图片等非
  文本块显示 `[image_url]` 占位.
- **Memory Trace 节点文字不再溢出卡片** — SVG 文本加裁剪, 长文案不会再溢出到相邻
  泳道.

### 兼容性

- 纯增量: 新端点 `POST /api/persona/import_from_archive` /
  `POST /api/memory/lineage/attribute_all`, 不改动既有端点契约与存档格式.

## v1.3.0 (2026-06-29) — P27 记忆溯源可视化 (Memory Trace)

新增**第 6 个 workspace「Memory Trace 记忆溯源」**: 一张只读的节点流水线图,
把当前角色的记忆按「对话 → recent 摘要 → 事实 → 反思 → 人设」从左到右分层画出来,
让你一眼看清一条记忆从哪来、到哪去. 灵感来自建模软件的 shader 节点图.

### 新增功能

- **Memory Trace workspace (顶栏第 6 tab)** — 纯 SVG 节点图 (无第三方图库):
  - **总览模式**: 看整体记忆构成; 点任意节点, 右栏展开其完整内容 + 上游来源 +
    下游影响 (可点链接跳转).
  - **聚焦模式**: 点「聚焦此节点」, 只高亮该记忆的全部上游与下游 (祖先 + 后代),
    其余淡化 — 用来追踪某一条特定记忆的生成历史.
  - **诚实分层的连线**: **实线 = 已落盘真因果** (反思的 `source_fact_ids` → 事实;
    人设的 `source_id` / `merged_from_ids` → 反思; 矛盾 `old_text` 命中 → 人设);
    **虚线 = 启发式推断** (见下).
  - 对**所有记忆类型可用**: 事实 / 反思 / 人设 / 人设矛盾 / recent 摘要 / 对话消息.

- **Tier C 反向归因 (按需, 虚线)** — 对一条事实 / 反思 / 人设记忆点「分析来源」,
  用对话语料 (`time_indexed.db` + `recent.json`) 反推它可能来自哪些对话:
  - **文本相似度** (默认, 免费, 确定性): 字符二元组重叠打分.
  - **LLM 精判** (可选): 走 memory 组模型, 让模型从相似度候选里挑真来源.
  - 结果一律画**虚线** + 置信分, 明确标注"启发式重建", 绝不冒充真因果.

- **第一个对话归档消费者** — testbench 此前从不读取主程序保存的逐轮对话原文
  (`time_indexed.db`), Memory Trace 是第一个解析并呈现它的功能. **从真实角色导入**
  时该归档已随记忆目录拷入沙盒, 因此导入的角色可获得对话级溯源.

### 已知限制

- **预设 / 未对话角色无对话级溯源**: 内置预设角色和从未对话过的角色没有
  `time_indexed.db`, 对话泳道为空, Tier C 归因也为空 (UI 会明示).
- **重度角色对话归档不随存档保留**: 单个 `time_indexed.db` 若超过存档单文件大小
  上限, 不会被「保存 / 加载存档」保留 (既有存档格式取舍, 未改动). 需要对话级溯源
  时请**重新从真实角色导入**, 不要依赖存档往返.
- **变迁史粗粒度**: testbench 原生记忆不产生 `events.ndjson` evidence 时间线,
  记忆变迁史只能由 JSON status 字段粗粒度重建.

### 兼容性

- 全部为**只读**功能: 不写记忆 JSON, 不改主程序, 不改持久化 schema / 导出格式 /
  默认端口 / 数据目录结构.
- 新增 HTTP 端点 `GET /api/memory/lineage` 与 `POST /api/memory/lineage/attribute`,
  不影响现有端点. smoke 套件 24 → 26 份 (+ UI smoke p33).

### 基线能力

- **会话与沙盒** — 单活跃会话 · 每会话一个隔离沙盒 · 主程序 config
  manager 单例约束下的强制切换.

  相对时间渲染 · 不影响真实墙钟.

  · 双 AI 自动对话.

  手动 Op (压缩最旧消息 / 从对话抽取事实 / 合成反思 / 修正 persona
  事实 / 加 persona 事实), 全部两阶段 (Preview / Commit).

  人员从 "创建会话" 到 "开测" 的最短路径.

  · Single-turn 单条打分 · Full-session 全轮评分, 带 ScoringSchema

  · 崩溃后启动时恢复上次会话.

  11 组合. 导出时 `api_key` 自动脱敏.

  Reset / Safety Audit.

  · Rewind.
