## 当前状态

- 产品重点仍是 Independent Mode：先让 NEKO 能在 10-50 人左右的独播直播间自然撑住约 30 分钟，不尬聊、不刷屏、不像模板主持。
- 主链路是 Live Ingest -> EventBus -> `live_events` Selection -> Pipeline -> Safety Guard -> `NekoDispatcher` -> Dashboard / Monitor projection。
- 直播专用发言治理已经收回插件侧：宿主核心不再承载 NEKO Live 专用的最终输出整形、抑制、重试或审计逻辑。
- Gift / SC / Guard 已走 `live_support_events` 短句致谢 / signal 路线，不能再掉回普通头像锐评或普通弹幕回复。
- 热梗库和冷场素材库已经变成可交接 JSON：`data/meme_knowledge.json` 当前 36 条，`data/idle_hosting_beats.json` 当前 32 条。

## 硬边界

- 不要把影响猫猫直播发言的专用逻辑塞回宿主核心；除非重新开架构评审。
- 会影响“猫猫要不要回这条弹幕 / 怎么接这条弹幕”的能力，必须属于插件内置功能，或沉淀成通用宿主接口。
- `activity_level` 是公开节奏旋钮，不新增和它重叠的 `reply_selection_mode`。`live_events` 只派生只读 `reply_selection_policy` 给 Dashboard / Monitor 复盘。
- `standard` / `active` 只跳过 `666`、纯反应、重复数字等低价值弹幕；`quiet` 会额外跳过低优先级普通短句，但问题、内容请求、问候、舰长 / 高分事件仍放行。
- `avatar_roast` 默认不能念裸 UID 数字。UID 是路由身份，除非昵称缺失或观众明确把 ID 当话题。
- `danmaku_response` 必须先接当前弹幕。近期上下文、观众画像、房间主题、热梗提示都只是 advisory。
- 不推进抖音 transport / 登录 / 签名，除非当前直播效果闭环已经复盘完。

## 接手顺序

1. 先跑一场真实独播复盘，不要先扩 transport。重点看回复延迟、长回复、低价值弹幕过滤、头像 / UID 过度锐评、冷场素材 family 复读。
2. 根据复盘先调两份 JSON 素材。坏 JSON、缺字段和重复 key 必须安全降级，不能让直播输出断掉。
3. 如果观测字段确实有用，再考虑 UI 只读展示。新增可见文案要同步 8 个 locale。
4. 继续做观众画像治理：单 UID 删除 / 印象重置体验、freshness 行为、watch_time / contribution_rank 等非 raw 指标。
5. 旧 `bilibili_danmaku` 最终退役单独拆 PR，不和直播发言质量、素材库或 UI 打磨混在一起。
